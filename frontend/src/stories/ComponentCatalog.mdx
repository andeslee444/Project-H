## Development Guidelines

### Creating New Components

1. **Plan the Component**
   - Define props interface
   - Consider accessibility requirements
   - Plan for responsive design
   - Consider HIPAA compliance if handling PHI

2. **Implement the Component**
   ```tsx
   // components/ui/MyComponent/MyComponent.tsx
   export const MyComponent: React.FC<MyComponentProps> = () => {
     // Implementation
   };
   ```

3. **Add TypeScript Types**
   ```tsx
   // components/ui/MyComponent/types.ts
   export interface MyComponentProps {
     // Props definition
   }
   ```

4. **Create Stories**
   ```tsx
   // components/ui/MyComponent/MyComponent.stories.tsx
   export default {
     title: 'Category/MyComponent',
     component: MyComponent,
   };
   ```

5. **Write Tests**
   ```tsx
   // components/ui/MyComponent/MyComponent.test.tsx
   describe('MyComponent', () => {
     // Tests
   });
   ```

### Component Checklist

Before marking a component as complete:

- [ ] TypeScript types defined
- [ ] Props documented with JSDoc
- [ ] Stories cover all variants
- [ ] Responsive design implemented
- [ ] Accessibility tested (keyboard, screen reader)
- [ ] Dark mode supported
- [ ] Unit tests written (>80% coverage)
- [ ] Error states handled
- [ ] Loading states implemented
- [ ] HIPAA compliance verified (if applicable)

## Contributing

To contribute a new component:

1. Check this catalog to avoid duplicates
2. Follow the component structure guidelines
3. Ensure all checklist items are complete
4. Submit PR with component, stories, and tests
5. Update this catalog with your component
